  <name>Task 1: Advisor auth helpers and data access layer</name>
  <files>
    src/lib/advisor/auth.ts
    src/lib/data/advisor.ts
  </files>
  <action>
1. **Create src/lib/advisor/auth.ts**:
   - `requireAdvisorRole()`: Calls `auth()` from @/lib/auth, checks session.user.role is 'ADVISOR' or 'ADMIN'. Throws "Unauthorized: Advisor access required" if not. Returns { userId, role }.
   - `getAdvisorProfileOrThrow(userId: string)`: Queries AdvisorProfile by userId. Throws "Advisor profile not found" if missing. Returns the profile with id.
   - Both functions use "server-only" import guard.

2. **Create src/lib/data/advisor.ts** with "server-only" import:

   - `getAdvisorProfile(userId: string)`: Find AdvisorProfile by userId, include user name/email.

   - `getAssignedClients(advisorProfileId: string)`: Query ClientAdvisorAssignment where advisorId = advisorProfileId AND status = ACTIVE. Include client User (id, name, email). For each client, fetch their latest IntakeInterview (most recent by updatedAt) with response count. Return as AdvisorDashboardClient[].

   - `getClientIntakeForReview(advisorProfileId: string, interviewId: string)`: Fetch IntakeInterview by id, but ONLY if the interview's userId has an ACTIVE ClientAdvisorAssignment to this advisor. Include all IntakeResponses (ordered by question sequence). Also fetch the IntakeApproval for this interview if one exists. This is the critical security boundary -- the query must join through assignment to verify the advisor has access to this client.

   - `createIntakeApproval(interviewId: string, advisorProfileId: string)`: Create IntakeApproval with status PENDING. Use upsert keyed on interviewId to prevent duplicates.

   - `updateIntakeApproval(approvalId: string, data: { status?, focusAreas?, notes?, reviewedAt?, approvedAt? })`: Update approval record. If status is APPROVED, set approvedAt to now(). If status is IN_REVIEW, set reviewedAt to now().

   - `getAdvisorNotifications(advisorProfileId: string, unreadOnly?: boolean)`: Fetch AdvisorNotification records, optionally filtered to unread. Order by createdAt desc. Limit 50.

   - `markNotificationRead(notificationId: string, advisorProfileId: string)`: Update notification read=true, but ONLY WHERE advisorId matches (ownership check).

   - `createNotification(advisorProfileId: string, type: NotificationType, title: string, message: string, referenceId?: string)`: Create AdvisorNotification record.
  </action>

  <name>Task 2: Authenticated server actions for advisor portal</name>
  <files>src/lib/actions/advisor-actions.ts</files>
  <action>
Create src/lib/actions/advisor-actions.ts with "use server" directive. Follow the exact same pattern as intake-actions.ts (try/catch, { success, error, data } response format).

Every action starts with:
```
const { userId } = await requireAdvisorRole();
const profile = await getAdvisorProfileOrThrow(userId);
```

Actions:

1. **getAdvisorDashboardData()**: Returns { clients: AdvisorDashboardClient[], profile: AdvisorProfile, unreadNotificationCount: number }. Uses getAssignedClients and getAdvisorNotifications (count unread).

2. **getIntakeReviewData(interviewId: string)**: Validates interviewId with z.string().cuid(). Gets intake via getClientIntakeForReview. Returns IntakeReviewData including interview with responses, approval state, and INTAKE_QUESTIONS array for mapping questionId to question text. Returns 404 error if interview not found or not assigned.

3. **markIntakeInReview(interviewId: string)**: Creates or updates IntakeApproval to IN_REVIEW status. Uses createIntakeApproval if none exists, then updateIntakeApproval to set status. Revalidates path.

4. **approveClientIntake(data: unknown)**: Validates with approveClientSchema. Updates IntakeApproval to APPROVED with focusAreas and notes. Revalidates path. Returns the updated approval.

5. **rejectClientIntake(approvalId: string, notes?: string)**: Updates IntakeApproval to REJECTED with notes. Revalidates path.

6. **getAdvisorNotificationsAction()**: Returns notifications list using getAdvisorNotifications.

7. **markNotificationReadAction(notificationId: string)**: Marks single notification as read.

All actions: wrap in try/catch, return { success: boolean, error?: string, data?: T }.
