<task type="auto">
  <name>Task 1: Prisma schema extensions for advisor portal</name>
  <files>prisma/schema.prisma</files>
  <action>
Add the following to the existing Prisma schema:

1. **UserRole enum** with values: USER, ADVISOR, ADMIN

2. **Add `role` field to User model**: `role UserRole @default(USER)` — add after `image` field. Also add relations: `advisorProfile AdvisorProfile?` and `clientAssignments ClientAdvisorAssignment[] @relation("ClientAssignments")` and `notifications AdvisorNotification[]`

3. **AdvisorProfile model**:
   - id: String @id @default(cuid())
   - userId: String @unique
   - user: User relation
   - specializations: String[] (risk area expertise, values from assessment subcategory IDs like 'decision-making-authority', 'succession-planning', etc.)
   - licenseNumber: String? (optional advisor credentials)
   - firmName: String? (optional firm name)
   - bio: String? (optional bio text)
   - createdAt: DateTime @default(now())
   - updatedAt: DateTime @updatedAt
   - clientAssignments: ClientAdvisorAssignment[] @relation("AdvisorAssignments")
   - Index on userId

4. **ClientAdvisorAssignment model**:
   - id: String @id @default(cuid())
   - clientId: String
   - advisorId: String (references AdvisorProfile.id, NOT User.id)
   - assignedAt: DateTime @default(now())
   - status: AssignmentStatus @default(ACTIVE)
   - client: User @relation("ClientAssignments")
   - advisor: AdvisorProfile @relation("AdvisorAssignments")
   - @@unique([clientId, advisorId])
   - Index on advisorId, index on clientId

5. **AssignmentStatus enum**: ACTIVE, INACTIVE

6. **IntakeApproval model** (tracks advisor approval of client intakes):
   - id: String @id @default(cuid())
   - interviewId: String @unique
   - advisorId: String
   - status: ApprovalStatus @default(PENDING)
   - focusAreas: String[] (selected subcategory IDs from assessment)
   - notes: String? (advisor notes)
   - reviewedAt: DateTime?
   - approvedAt: DateTime?
   - createdAt: DateTime @default(now())
   - updatedAt: DateTime @updatedAt
   - interview: IntakeInterview relation
   - advisor: AdvisorProfile relation
   - Index on advisorId, index on interviewId

7. **ApprovalStatus enum**: PENDING, IN_REVIEW, APPROVED, REJECTED

8. **AdvisorNotification model**:
   - id: String @id @default(cuid())
   - advisorId: String
   - type: NotificationType
   - title: String
   - message: String
   - referenceId: String? (e.g., interviewId for intake notifications)
   - read: Boolean @default(false)
   - createdAt: DateTime @default(now())
   - advisor: AdvisorProfile relation
   - Index on [advisorId, read]

9. **NotificationType enum**: NEW_INTAKE, INTAKE_UPDATED, SYSTEM

Add the IntakeApproval relation back-reference to IntakeInterview model: `approval IntakeApproval?`
Add AdvisorNotification[] relation to AdvisorProfile.

Run `npx prisma db push` to apply.
