<task type="auto">
  <name>Task 2: Create invitation types and business logic service</name>
  <files>src/lib/invitations/types.ts, src/lib/invitations/service.ts, src/lib/invite.ts</files>
  <action>
**src/lib/invitations/types.ts:**
Define TypeScript types:
- `CreateInvitationInput`: { clientEmail: string, clientName?: string, personalMessage?: string }
- `InvitationWithDetails`: Full invitation record with advisor info, computed properties (isExpired, canResend)
- `InvitationListFilters`: { status?: InvitationStatus, search?: string }
- Re-export InvitationStatus from Prisma client

**src/lib/invite.ts:**
Add a new exported constant `INVITATION_TTL_SEC = 60 * 60 * 24 * 7` (7 days) for client invitations. Keep the existing `INVITE_TOKEN_TTL_SEC` (10 minutes) for backward compatibility with the existing code-based flow. Add a new `createInvitationToken(inviteCodeId: string)` function that uses the 7-day TTL, following the same HMAC-SHA256 pattern as `createInviteToken`. The existing `verifyInviteToken` already handles both since it reads expiry from the token itself.

**src/lib/invitations/service.ts:**
Mark with `"server-only"` import. Create functions:

1. `createAdvisorInvitation(advisorId: string, input: CreateInvitationInput)`:
   - Use `requireAdvisorRole()` pattern from existing code is NOT needed here (caller handles auth)
   - Generate a random 6-character alphanumeric code (uppercase) using crypto.randomBytes
   - Create InviteCode record in a transaction: set code, prefillEmail=clientEmail, expiresAt=now+7days, maxUses=1, createdBy=advisorId, status=SENT, personalMessage, clientName
   - Generate secure token using new `createInvitationToken`
   - Build invitation URL: `${NEXTAUTH_URL}/signup?invite=${token}&callbackUrl=/intake`
   - Return the created invitation with the URL (URL is needed for email sending in Plan 02)

2. `getAdvisorInvitations(advisorId: string, filters?: InvitationListFilters)`:
   - Query inviteCode where createdBy=advisorId
   - Apply optional status and search (email/name) filters
   - Order by createdAt desc
   - Map results to InvitationWithDetails (compute isExpired from expiresAt, canResend from status)

3. `resendInvitation(advisorId: string, invitationId: string)`:
   - Find invitation, verify it belongs to advisor (multi-tenant security)
   - Check resend limit: max 3 resends (usedCount tracks sends, not registrations - repurpose or add sendCount field)
   - Actually: add `resendCount Int @default(0)` to schema if not there. If you cannot modify schema in this task, track via usedCount field with comment explaining dual use.
   - Reset expiresAt to now+7days, update status back to SENT
   - Generate new token and URL
   - Return updated invitation with new URL

4. `expireInvitation(advisorId: string, invitationId: string)`:
   - Find invitation, verify ownership
   - Update status to EXPIRED, set expiresAt to now
   - Return updated record

Note: If `resendCount` field is needed, add it to schema.prisma in this task alongside the other changes.
