<task type="auto">
  <name>Task 2: Create advisor-scoped analytics queries and chart container component</name>
  <files>src/lib/analytics/queries.ts, src/components/analytics/ChartContainer.tsx</files>
  <action>
    1. Create `src/lib/analytics/queries.ts` with `"server-only"` guard at top (same pattern as `src/lib/dashboard/queries.ts`):

       `getFamilyGovernanceTrends(clientId: string, advisorProfileId: string): Promise<FamilyAnalyticsData>`
       - Query `prisma.clientAdvisorAssignment.findFirst` to verify advisor-client relationship (where advisorId = advisorProfileId, clientId = clientId, status = 'ACTIVE'). Throw error if not found (security enforcement).
       - Query `prisma.assessment.findMany` for the client where status = 'COMPLETED', ordered by completedAt ASC, including `scores` (PillarScore records) and selecting completedAt, id.
       - For each assessment, compute overallScore as weighted average of PillarScore records using the weights from the pillar definition (decision-making: 20, access-controls: 15, trust-estate: 15, marriage-risk: 10, succession: 15, behavior: 10, business: 10, documentation: 5). Total weight = 100.
       - Build `trendData` array: map each assessment to `{ date: assessment.completedAt.toISOString(), overallScore: computedWeightedAvg, assessmentId: assessment.id }`
       - Build `assessments` array: for each assessment, extract CategoryBreakdownPoint[] from PillarScore records (pillar field maps to categoryId, use CATEGORY_LABELS for name, score from PillarScore.score). Calculate trendDirection by comparing overallScore to previous assessment (first = 'new', higher = 'improving', lower = 'declining', equal = 'stable'). Threshold for improving/declining: difference > 0.3 points.
       - Build `latestBreakdown` from the most recent assessment's PillarScore records.
       - Get client name from the assignment query include.
       - Return complete FamilyAnalyticsData object.

       `getAssessmentComparison(assessmentId: string, advisorProfileId: string): Promise<AssessmentComparison>`
       - Fetch single assessment by ID with PillarScore includes.
       - Verify advisor has access via clientAdvisorAssignment check on the assessment's userId.
       - Return AssessmentComparison with category breakdown.

       - Props: `{ title: string; description?: string; children: ReactNode; height?: number; className?: string }`
       - Renders a Card (from @/components/ui/card) with CardHeader containing title and optional description, CardContent wrapping children in Recharts `ResponsiveContainer` with width="100%" and configurable height (default 300).
       - Follow the hero-surface/Card pattern from the existing dashboard page.
       - This is a server component (no "use client" needed since ResponsiveContainer handles client-side rendering internally — actually, ResponsiveContainer requires client rendering). Mark as `"use client"` component.
  </action>
