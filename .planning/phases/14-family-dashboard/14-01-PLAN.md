<task type="auto">
  <name>Task 1: Create family dashboard types and data query</name>
  <files>src/lib/family/types.ts, src/lib/family/queries.ts</files>
  <action>
Create `src/lib/family/types.ts` with these interfaces:

- `FamilyHouseholdMember`: { fullName: string; relationship: string; governanceRoles: string[] }
- `FamilyPillarScore`: { pillar: string; pillarName: string; score: number; weight: number; riskLevel: string }
- `FamilyHistoricalAssessment`: { assessmentId: string; completedAt: string; overallScore: number; pillarScores: FamilyPillarScore[]; trendDirection: 'improving' | 'declining' | 'stable' | 'new' }
- `FamilyDashboardData`: { householdMembers: FamilyHouseholdMember[]; currentScore: number | null; currentPillarScores: FamilyPillarScore[]; historicalAssessments: FamilyHistoricalAssessment[]; advisorEmphasis: string[]; hasMultipleAssessments: boolean }

Create `src/lib/family/queries.ts` with `"server-only"` guard at top:

- Import PILLAR_WEIGHTS from `@/lib/analytics/queries` and CATEGORY_LABELS from `@/lib/analytics/formatters`
- Import prisma from `@/lib/db`
- Export `calculateWeightedScore(pillarScores)` pure function (same pattern as analytics/queries.ts)
- Export `getTrendDirection(current, previous)` returning trend direction with 0.3 threshold (same as analytics)
- Export `getFamilyDashboardData(userId: string): Promise<FamilyDashboardData>`:
  - Query `prisma.user.findUnique` with userId
  - Include `householdMembers` (select fullName, relationship, governanceRoles)
  - Include `assessments` where status is 'COMPLETED', ordered by completedAt desc, including `scores` relation
  - To get advisor emphasis: include user's `intakeInterviews` with `approval` relation (IntakeApproval has focusAreas string array)
  - Map household members to FamilyHouseholdMember
  - For each completed assessment, calculate weighted overall score, map pillar scores with CATEGORY_LABELS for names and PILLAR_WEIGHTS for weights, determine trend direction vs previous assessment
  - Extract focusAreas from the latest IntakeApproval (if exists)
  - Return FamilyDashboardData with currentScore from latest assessment, currentPillarScores, all historical assessments, and advisorEmphasis
  </action>
  <verify>Run `npx tsc --noEmit` — zero type errors. Verify both files exist and exports are importable.</verify>
  <done>Family types defined with all required interfaces. Query function loads household members, all completed assessments with pillar scores, calculates weighted scores and trend directions, and extracts advisor focus areas from IntakeApproval.</done>
