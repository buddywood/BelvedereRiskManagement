**Create `src/app/(protected)/advisor/unified-risk/page.tsx`:**

Follow the Suspense streaming pattern from cyber-risk/page.tsx but with unified data:

Layout structure:
1. **Back to Portal link** (ArrowLeft icon, links to /advisor)
2. **Metrics row** (5 cards in grid): Total Clients, Fully Assessed (all 3 pillars complete), Average Composite Score (X.X / 10), At Risk (HIGH/CRITICAL composite), Assessment Coverage (e.g., "67% of pillars")
3. **Client table** inside Card with title "Unified Risk Portfolio":
   - Columns: Client Name, Email, Composite Score, Governance, Cyber Risk, Identity Risk, Risk Level
   - Composite Score column shows weighted composite (X.X / 10) or "Incomplete" if no pillars assessed
   - Individual pillar columns show score (X.X) or dash (--) if not assessed, using muted text for unassessed
   - Risk Level column uses Badge with same variant mapping as cyber-risk dashboard (CRITICAL=red, HIGH=amber, MEDIUM=secondary, LOW=success)
   - Sort by composite score ascending (highest risk first), unassessed clients last
4. **Empty state** when no clients have any assessments

Use Suspense boundary with UnifiedRiskLoading as fallback. Import loading component from ./loading.

Key UI details:
- Each individual pillar score cell should show a small colored dot indicator: green (>=7.5), amber (>=5.0), red (>=2.5), dark red (<2.5) to quickly visualize which domains need attention
- "Fully Assessed" metric counts clients where all 3 pillars have scores (pillarsAssessed === 3)
- "Assessment Coverage" = total pillar assessments completed / (totalClients * 3) as percentage
