  <name>Task 1: Unified risk types and composite scoring engine</name>
  <files>src/lib/unified-risk/types.ts, src/lib/unified-risk/scoring.ts</files>
  <action>
Create `src/lib/unified-risk/types.ts`:

Define types for the unified risk system:
- `PillarSummary`: `{ pillar: string; pillarLabel: string; score: number; riskLevel: RiskLevel; assessedAt: string | null; }` -- represents one pillar's latest score
- `CompositeScore`: `{ score: number; riskLevel: string; pillarScores: PillarSummary[]; pillarsAssessed: number; totalPillars: number; }` -- the weighted composite
- `UnifiedRiskProfile`: `{ clientId: string; clientName: string; clientEmail: string; composite: CompositeScore; governance: PillarSummary | null; cyberRisk: PillarSummary | null; identityRisk: PillarSummary | null; }` -- full client profile
- `UnifiedPortfolioData`: `{ totalClients: number; assessedClients: number; averageComposite: number | null; clientsAtRisk: number; profiles: UnifiedRiskProfile[]; }` -- portfolio aggregate
- `ActionPlanItem`: `{ id: string; domain: 'governance' | 'cyber-risk' | 'identity-risk'; title: string; description: string; priority: 'high' | 'medium' | 'low'; status: 'pending' | 'in-progress' | 'completed'; }` -- for UNIFIED-02 action plans
- Export `DOMAIN_WEIGHTS` constant: `{ 'family-governance': 40, 'cyber-risk': 30, 'identity-risk': 30 }` -- governance weighted higher as primary pillar
- Export `PILLAR_LABELS` constant: `{ 'family-governance': 'Family Governance', 'cyber-risk': 'Cyber Risk', 'identity-risk': 'Identity Risk' }`
- Import `RiskLevel` from `@prisma/client`

Create `src/lib/unified-risk/scoring.ts`:

Implement `calculateCompositeScore(pillarScores: PillarSummary[]): CompositeScore`:
- Takes array of PillarSummary objects (only assessed pillars, 0-3 items)
- Calculates weighted average using DOMAIN_WEIGHTS (only for pillars present in input)
- If no pillars assessed, return score 0 with pillarsAssessed: 0
- Use same `getRiskLevel` function from `../../assessment/scoring` to classify composite score
- Return CompositeScore with breakdown of which pillars contributed
- totalPillars is always 3 (governance + cyber + identity)

Key: This mirrors the existing `calculateWeightedScore` pattern in `src/lib/analytics/queries.ts` but operates on the three risk domains rather than governance sub-categories.

- Verify advisor-client assignment (same pattern as `getTopRisksForFamily` in intelligence/queries.ts)
- Fetch latest completed assessment for the client
- Include all PillarScore records (no pillar filter)
- For each of the three known pillars ('family-governance', 'cyber-risk', 'identity-risk'), extract the PillarSummary or null
- Call `calculateCompositeScore` with the non-null pillar summaries
- Return UnifiedRiskProfile

**`getUnifiedPortfolioData(advisorProfileId: string): Promise<UnifiedPortfolioData>`**
- Get all active client assignments for advisor (same pattern as getCyberRiskDashboardData in advisor-actions.ts)
- For each client, fetch latest completed assessment with ALL pillar scores
- Build UnifiedRiskProfile for each client
- Calculate portfolio aggregates: totalClients, assessedClients (clients with at least 1 pillar scored), averageComposite (of assessed clients), clientsAtRisk (composite riskLevel HIGH or CRITICAL)
- Sort profiles by composite score ascending (highest risk first)
- Return UnifiedPortfolioData
