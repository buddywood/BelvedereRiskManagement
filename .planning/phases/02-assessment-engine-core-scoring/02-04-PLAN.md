  <name>Task 1: Score calculation API endpoint with pillar completion logic</name>
  <files>
    src/app/api/assessment/[id]/score/route.ts
  </files>
  <action>
    Create POST endpoint at `/api/assessment/[id]/score` that:

    1. Authenticates user via `auth()` session
    2. Loads assessment by ID, verifies ownership (403 if not owner)
    3. Loads all AssessmentResponse records for the assessment
    4. Converts responses to answers Record (questionId -> answer value)
    5. Calls `calculatePillarScore(answers, familyGovernancePillar, allQuestions)` from scoring.ts
    6. Calls `identifyMissingControls(answers, allQuestions)` from scoring.ts
    7. Upserts PillarScore record: `prisma.pillarScore.upsert()` with `@@unique([assessmentId, pillar])`
       - Store: score, riskLevel enum, breakdown as JSON, missingControls as JSON, calculatedAt
    8. Updates Assessment status to COMPLETED, sets completedAt timestamp
    9. Returns: `{ score, riskLevel, breakdown, missingControls, completedAt }`

    Also create a GET variant for `/api/assessment/[id]/score` that:
    1. Loads existing PillarScore for the assessment
    2. Returns cached score data (no recalculation unless POST is called)
    3. Returns 404 if no score exists yet

    The POST endpoint is called when the user completes the last question and clicks "Complete Section". The GET is used when revisiting results.

    Map RiskLevel string from scoring.ts to Prisma's RiskLevel enum: 'low' -> LOW, 'medium' -> MEDIUM, 'high' -> HIGH, 'critical' -> CRITICAL.

    Error handling: If assessment has fewer than 50% of questions answered, return 400 with message explaining incomplete assessment cannot be scored (per user decision: unanswered never defaults to low risk; better to block scoring than give misleading score).
  </action>
  <verify>
    - `npm run build` passes
    - API route registered in build output
    - POST endpoint calculates and persists score
    - GET endpoint returns cached score
    - Score calculation excludes unanswered questions
    - Incomplete assessments (< 50% answered) are rejected
  </verify>
  <done>
    Score API endpoint calculates weighted pillar score from assessment responses, identifies missing controls, persists results in PillarScore table, and marks assessment as completed.
  </done>
</task>
