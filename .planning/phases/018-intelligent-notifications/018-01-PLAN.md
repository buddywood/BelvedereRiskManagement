   - WORKFLOW_STALLED (NOTIFY-04: stalled workflow escalation)
   - DOCUMENT_UPLOADED (NOTIFY-03: client uploaded document)
   Keep existing values: NEW_INTAKE, INTAKE_UPDATED, NEW_LEAD, SYSTEM.

2. Add NotificationPreference model to prisma/schema.prisma:
   ```
   model NotificationPreference {
     id        String   @id @default(cuid())
     userId    String
     user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)

     // Email notification toggles
     emailEnabled          Boolean @default(true)
     emailMilestones       Boolean @default(true)   // NOTIFY-03: milestone completions
     emailReminders        Boolean @default(true)    // NOTIFY-02: assessment reminders
     emailStalled          Boolean @default(true)    // NOTIFY-04: stalled workflow alerts
     emailRegistrations    Boolean @default(true)    // NOTIFY-01: client registrations

     // Frequency control
     reminderFrequencyDays Int     @default(7)       // Min days between reminder emails

     // Quiet hours (optional, store as "HH:MM" strings, UTC)
     quietStart   String?  // e.g., "22:00"
     quietEnd     String?  // e.g., "08:00"

     createdAt DateTime @default(now())
     updatedAt DateTime @updatedAt

     @@unique([userId])
   }
   ```
   Add the `notificationPreferences` relation to the User model: `notificationPreference NotificationPreference?`

3. Run `npx prisma db push` to apply schema changes.

4. Create src/lib/notifications/types.ts with TypeScript types:

   - SendNotificationParams interface: { recipientUserId, recipientEmail, category, title, message, referenceId?, advisorProfileId? (for in-app), emailSubject?, emailHtml? }
   - Map from NotificationCategory to the preference field name (e.g., 'registration' -> 'emailRegistrations')
