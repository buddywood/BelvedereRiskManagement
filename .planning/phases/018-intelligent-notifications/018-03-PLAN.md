  <files>src/app/api/cron/workflow-reminders/route.ts, src/lib/reminders/workflow-reminders.ts, src/lib/reminders/assessment-reminders.ts</files>
  <action>
1. Create src/lib/reminders/assessment-reminders.ts (server-only):
   - processAssessmentReminders(): Finds clients with incomplete assessments/intakes that have been inactive
   - Query logic:
     a. Find ClientAdvisorAssignment records where client has an IntakeInterview with status IN_PROGRESS and updatedAt older than 7 days, OR Assessment with status IN_PROGRESS and updatedAt older than 14 days
     b. For each client, check shouldSendNotification(clientUserId, 'reminder', 'email') from preferences
     c. Check deduplication: only send if no reminder sent in last N days (use user's reminderFrequencyDays preference, default 7)
     d. Track last reminder sent via a simple approach: query AdvisorNotification for SYSTEM type with referenceId matching clientId within the frequency window. OR add a lastAssessmentReminderSentAt field -- prefer using the notification record approach to avoid schema changes.
   - For each eligible client, call sendNotification with category 'reminder', sending to the CLIENT (not advisor)
   - Use assessment reminder email template from templates.ts
   - Return { clientsReminded: number }

2. Create src/lib/reminders/workflow-reminders.ts (server-only):
   - processWorkflowReminders(): Finds stalled workflows and notifies ADVISORS
   - Query logic:
     a. Use the existing isStalled() function from src/lib/pipeline/status.ts (>7 days inactive)
     b. Find all ClientAdvisorAssignment records, compute stage for each client using computeClientStage()
     c. Filter to clients where isStalled(lastActivity, stage) is true AND stage is not COMPLETE
     d. For clients stalled >30 days, escalate: set category to 'stalled'
     e. Check shouldSendNotification for advisor before sending
     f. Deduplication: skip if advisor already received a stalled notification for this client within 7 days (query existing AdvisorNotification records)
   - Call sendNotification for each stalled client, targeting the advisor
   - Return { advisorsNotified: number, clientsEscalated: number }

3. Create src/app/api/cron/workflow-reminders/route.ts:
   - Follow exact same pattern as src/app/api/cron/document-reminders/route.ts
   - GET endpoint with CRON_SECRET Bearer token auth
   - Call processAssessmentReminders() and processWorkflowReminders() sequentially
   - Return combined results: { success, assessmentReminders: { clientsReminded }, workflowReminders: { advisorsNotified, clientsEscalated }, processingTimeMs, timestamp }
