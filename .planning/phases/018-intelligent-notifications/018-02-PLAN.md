<task type="auto">
  <name>Task 1: Create notification trigger functions</name>
  <files>src/lib/notifications/triggers.ts</files>
  <action>
Create src/lib/notifications/triggers.ts (server-only) with these trigger functions:

1. triggerRegistrationNotification(clientUserId: string, clientName: string, clientEmail: string):
   - Look up the advisor linked to this client via ClientAdvisorAssignment (or InviteCode.createdBy if assignment doesn't exist yet)
   - If advisor found, call sendNotification with:
     - category: 'registration'
     - title: "New Client Registered"
     - message: "{clientName} ({clientEmail}) has registered from your invitation"
     - advisorProfileId for in-app notification
     - referenceId: clientUserId
   - Fire-and-forget pattern: wrap in try/catch, log errors but don't throw (notification failure must not block registration)

2. triggerMilestoneNotification(clientUserId: string, milestone: string):
   - Look up the advisor linked to this client via ClientAdvisorAssignment
   - Resolve client name from User record
   - Call sendNotification with:
     - category: 'milestone'
     - title: milestone label (e.g., "Intake Complete", "Assessment Complete")
     - message: "{clientName} has completed {milestone}"
     - advisorProfileId for in-app notification
     - referenceId: clientUserId
   - Same fire-and-forget pattern

3. triggerDocumentUploadNotification(clientUserId: string, documentName: string):
   - Look up advisor via ClientAdvisorAssignment
   - Call sendNotification with:
     - category: 'milestone'
     - title: "Document Uploaded"
     - message: "{clientName} uploaded {documentName}"
     - advisorProfileId for in-app notification
     - referenceId: clientUserId

All functions must:
- Use prisma queries to resolve advisor from client relationship
- Handle case where no advisor is assigned (no-op, log warning)
- Never throw -- notification failures are non-blocking
  </action>
