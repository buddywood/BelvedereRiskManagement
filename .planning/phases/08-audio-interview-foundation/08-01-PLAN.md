    - submitIntakeInterview(interviewId: string): Validates all questions have responses, marks as SUBMITTED
    - getIntakeInterview(interviewId: string): Returns interview with responses for current user

<task type="auto">
  <name>Task 3: Audio upload and transcription API routes</name>
  <files>
    src/app/api/intake/[id]/audio/route.ts
    src/app/api/intake/[id]/transcribe/route.ts
  </files>
  <action>
    1. Create src/app/api/intake/[id]/audio/route.ts:
    - POST handler accepting multipart FormData with fields: audio (Blob), questionId (string)
    - Authenticate via auth() session check (same pattern as existing API routes)
    - Validate interview ownership (userId match)
    - Store audio file to local filesystem: `public/uploads/intake/{interviewId}/{questionId}.webm` (simple local storage for MVP; cloud storage is a future concern)
    - Create uploads directory if not exists
    - Save response record with audioUrl pointing to the stored file
    - Return { success: true, audioUrl, responseId }
    - Handle errors: 401 for no auth, 404 for interview not found, 400 for missing audio/questionId, 500 for server errors
    - Add `public/uploads/` to .gitignore if not already present

    2. Create src/app/api/intake/[id]/transcribe/route.ts:
    - POST handler accepting JSON { questionId: string }
    - Authenticate and validate ownership
    - Look up the IntakeResponse for the questionId to get the audioUrl
    - Update transcriptionStatus to PROCESSING
    - Read the audio file from disk
    - Send to OpenAI Whisper API: POST https://api.openai.com/v1/audio/transcriptions with FormData containing file blob and model "whisper-1"
    - Use process.env.OPENAI_API_KEY for auth header
    - On success: update IntakeResponse with transcription text and status COMPLETED
    - On failure: update status to FAILED, log error
    - Return { success: true, transcription } or { success: false, error }
    - Note: This is called automatically after audio upload completes (the UI in Plan 03 will chain these calls)
  </action>

  </verify>
