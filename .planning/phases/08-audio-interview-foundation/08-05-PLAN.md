    - If no active interview: show brief intro page explaining the intake interview
      - Title: "Family Governance Intake Interview"

      - "Begin Interview" button that calls startIntakeInterview server action and redirects to /intake/interview

    2. Create src/app/(protected)/intake/interview/page.tsx:
    - "use client" component -- the main interview experience
    - On mount: load interview via server action, initialize store with interviewId
    - Compose: StepIndicator (top), QuestionDisplay (center), AudioRecorder (below question)
    - **One question per screen** per locked decision -- only current question visible
    - Navigation: Previous/Next buttons at bottom. Next button disabled until current question has a recording.
    - **Auto-save flow** (per locked decision "auto-save after each question"):
      1. User records audio -> onRecordingComplete receives blob
      2. Upload blob via fetch POST to /api/intake/{id}/audio with FormData (audio blob + questionId)
      3. On upload success: trigger transcription via fetch POST to /api/intake/{id}/transcribe with { questionId }
      4. Update store with response data
      5. When user clicks Next: store already saved, just advance currentQuestionIndex and call updateProgress server action
    - **Immediate submission** (per locked decision "automatically submit once final question is answered"):
      - On the last question, after recording completes and uploads successfully, automatically call submitIntakeInterview server action
      - Show brief "Submitting..." state, then redirect to /intake/complete
      - The Next button on last question is replaced with visual indication that recording the response will complete the interview
    - Loading state: skeleton while interview loads
    - Error state: toast on save/upload failures with retry option

    3. Create src/app/(protected)/intake/complete/page.tsx:
    - Server component showing completion confirmation
    - Per locked decision "next steps explanation: show what happens next including advisor review process and timeline expectations":
      - Large checkmark or success icon
      - Title: "Interview Complete"
      - Explanation: "Thank you for completing your intake interview. Here's what happens next:"
      - Steps: (1) Your advisor will review your audio responses and transcriptions, (2) They will identify key focus areas for your governance assessment, (3) You will receive a customized assessment based on their review
      - Timeline expectation: "Your advisor will typically review within 1-2 business days."
      - "Return to Dashboard" button linking to /dashboard

    4. Update src/app/(protected)/layout.tsx:
    - Add "Intake" navigation link to the nav grid, linking to /intake
    - Place it logically in the nav order (after Dashboard, before Assessment)
