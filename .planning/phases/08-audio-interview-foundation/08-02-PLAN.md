  <name>Task 1: Audio recording hook with cross-browser support</name>
  <files>src/lib/hooks/useAudioRecorder.ts</files>
  <action>
    Create useAudioRecorder hook implementing MediaRecorder API with cross-browser format detection.

    **Exports:** useAudioRecorder() returning:
    - isRecording: boolean
    - isPaused: boolean
    - duration: number (seconds elapsed, updated via interval)
    - audioBlob: Blob | null (available after stop)
    - audioUrl: string | null (object URL for playback preview)
    - error: string | null
    - startRecording(): Promise<void>
    - stopRecording(): void
    - resetRecording(): void (clears blob, URL, resets state)
    - hasPermission: boolean | null (null = not yet requested)

    **Implementation details:**
    - getPreferredMimeType() helper: tries ['audio/webm;codecs=opus', 'audio/mp4;codecs=mp4a.40.2', 'audio/webm', 'audio/mp4'] via MediaRecorder.isTypeSupported(), falls back to empty string
    - startRecording: request getUserMedia({ audio: true }), create MediaRecorder with detected MIME type, collect chunks via ondataavailable, start duration timer (setInterval every 1s)
    - stopRecording: stop MediaRecorder, clear duration interval, combine chunks into Blob, create object URL via URL.createObjectURL
    - resetRecording: revoke object URL (URL.revokeObjectURL to prevent memory leaks per research pitfall #4), clear blob and URL state
    - Error handling: catch NotAllowedError (permission denied), NotFoundError (no microphone), generic errors. Set error state with user-friendly messages.
    - Cleanup on unmount: stop any active recording, revoke object URLs, clear intervals (useEffect cleanup)
    - Do NOT use any external libraries -- MediaRecorder API is native and sufficient per research recommendation
  </action>
  <verify>
    Run `npx tsc --noEmit` compiles without errors.
    Verify hook handles: start, stop, reset, error states, cleanup on unmount.
    Verify getPreferredMimeType checks multiple formats for Safari compatibility.
  </verify>
  <done>
    useAudioRecorder hook provides full recording lifecycle with cross-browser MIME detection, duration tracking, blob output, and proper cleanup. No external audio dependencies.
  </done>

    1. Create src/lib/intake/store.ts following the pattern in src/lib/assessment/store.ts:

    **Zustand store with persist middleware** (localStorage key: "intake-interview"):
    - State: interviewId (string | null), currentQuestionIndex (number), responses (Record<string, { audioUrl?: string, transcription?: string, status: string }>), status (string), startedAt (string | null)
    - Actions: setInterviewId, setCurrentQuestion, setResponse (questionId, data), setStatus, reset
    - Computed: totalAnswered (count of responses with audioUrl), isComplete (totalAnswered === TOTAL_QUESTIONS)

    2. Create src/lib/hooks/useIntakeInterview.ts:

    **Orchestration hook** that combines store state with navigation logic:
    - Takes interviewId as parameter
    - Returns: currentQuestion (from INTAKE_QUESTIONS[index]), currentIndex, totalQuestions, progress (percentage), canGoNext, canGoPrev, goToNext(), goToPrev(), isFirstQuestion, isLastQuestion, getResponseForQuestion(questionId)

    - goToNext(): increments currentQuestionIndex in store. If on last question and it has a response, does NOT auto-submit (the UI component in Plan 03 handles submission). Simply prevents going past the last question.
    - goToPrev(): decrements currentQuestionIndex, minimum 0
    - progress: Math.round((currentIndex / totalQuestions) * 100)
    - canGoNext: currentIndex < totalQuestions - 1 OR (isLastQuestion AND current question has response -- signals ready to submit)
    - canGoPrev: currentIndex > 0

    Follow the hook pattern from useAssessmentNavigation.ts for consistency.
  </action>
