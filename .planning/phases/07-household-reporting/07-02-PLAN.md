  - `householdMembers`: Map all members to `{ fullName, relationship (title-cased), governanceRoles (title-cased array) }`
  - `decisionMakers`: Filter members with DECISION_MAKER role, extract fullName array
  - `successors`: Filter members with SUCCESSOR role, extract fullName array
  - `trustees`: Filter members with TRUSTEE role, extract fullName array
  - `advisors`: Filter members with ADVISOR role, extract fullName array
  - `beneficiaries`: Filter members with BENEFICIARY role, extract fullName array
  - `executors`: Filter members with EXECUTOR role, extract fullName array
  - `householdHead`: First DECISION_MAKER member name, or first member name as fallback
- Create a helper function `formatEnumValue(value: string): string` that converts SCREAMING_SNAKE to Title Case (split on underscore, lowercase, capitalize first letter of each word)
- When householdProfile is null/undefined, omit household fields entirely (let nullGetter handle in templates)

**generator.ts updates:**
- Add `nullGetter` configuration to Docxtemplater constructor options:
  ```
  nullGetter(part: any) {
    // Return empty string for missing household placeholders instead of throwing
    if (part.module === 'loop' || part.value?.startsWith('household') ||
        part.value?.startsWith('decisionMaker') || part.value?.startsWith('successor') ||
        part.value?.startsWith('trustee') || part.value?.startsWith('advisor') ||
        part.value?.startsWith('beneficiar') || part.value?.startsWith('executor') ||
        part.value?.startsWith('householdHead')) {
      return '';
    }
    // For non-household placeholders, preserve existing behavior
    return `{${part.value}}`;
  }
  ```
- This prevents template corruption when household data is absent (Pitfall 2 from research)
