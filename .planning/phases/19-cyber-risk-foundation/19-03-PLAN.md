  <name>Task 1: Create AI recommendation engine and API</name>
  <files>src/lib/cyber-risk/recommendations.ts, src/app/api/cyber-risk/recommendations/route.ts</files>
  <action>
**Create `src/lib/cyber-risk/recommendations.ts`:**
- Import `getOpenAIClient` from `@/lib/openai`
- Import `ScoreResult` from `@/lib/assessment/types`

Create `generateCyberRecommendations(scoreResult: ScoreResult, answers: Record<string, unknown>): Promise<CyberRecommendation[]>` where `CyberRecommendation = { title: string; description: string; priority: 'high' | 'medium' | 'low'; category: string }`.

Implementation:
- Build a structured prompt that includes:
  - Overall cyber risk score and risk level
  - Category breakdown scores (which sub-categories scored lowest)
  - Missing controls identified by the scoring engine
  - Key answer patterns (e.g., "user does not use MFA", "user uses SMS for MFA")
- Use `gpt-4o-mini` model (cost-effective for structured recommendations)
- System prompt: "You are a cybersecurity advisor for high-net-worth families. Generate specific, actionable recommendations based on their cyber risk assessment. Focus on practical steps, not technical jargon. Each recommendation should be something the family can implement within a week."
- Request 3-5 recommendations as JSON array
- Parse response with error handling (fallback to generic recommendations based on missingControls if API fails)
- Include a `generateFallbackRecommendations(scoreResult: ScoreResult): CyberRecommendation[]` function that maps missingControls to static recommendations as backup

Export `generateCyberRecommendations`, `CyberRecommendation` type.

**Create `src/app/api/cyber-risk/recommendations/route.ts`:**
- POST endpoint: accepts `{ assessmentId: string }` in body
- Auth check: verify session, verify assessment ownership (same pattern as score route)
- Load PillarScore for pillar='cyber-risk' from the assessment
- Load AssessmentResponses for pillar='cyber-risk'
- Convert responses to answers record
- Build ScoreResult from PillarScore data
- Call `generateCyberRecommendations(scoreResult, answers)`
- Return `{ recommendations: CyberRecommendation[] }`
- GET endpoint: same auth/ownership check, returns cached recommendations if previously generated (store in PillarScore.missingControls JSON field as `{ controls: [...], recommendations: [...] }`)
