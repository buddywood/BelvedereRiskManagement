<objective>
Enhance branching engine to support answer change re-evaluation, orphaned data handling, and branching-aware scoring.

Purpose: The existing simple branching works for initial question flow but does not handle the critical case where a user changes a gate answer (e.g., changes "has trusts" from yes to no). When this happens, previously-visible questions become hidden, their answers become orphaned, and scoring/completion calculations must adjust. This plan builds the engine logic; Plan 02 wires it into the UI.

Output: Enhanced branching.ts with change detection and orphan identification, branching-aware score API, comprehensive test suite covering all path combinations.
</objective>
