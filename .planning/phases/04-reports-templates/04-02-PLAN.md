    2. Create `src/lib/templates/types.ts`:
       - `TemplateId` union type: 'decision-making-authority' | 'access-controls' | 'trust-estate-governance' | 'succession-planning' | 'behavior-standards' | 'family-business-governance' | 'documentation-records'
       - `TemplateMetadata` interface: { id: TemplateId, name: string, description: string, category: string, applicableSubCategories: string[] }
       - `TemplateData` interface: { familyName: string, assessmentDate: string, overallScore: number, riskLevel: string, categoryScore: number, categoryRiskLevel: string, gaps: Array<{ description: string, severity: string, recommendation: string }>, strengths: string[], recommendations: string[] }
       - `TEMPLATE_REGISTRY` const array of all 7 TemplateMetadata entries, mapping each template to its corresponding assessment sub-category

    3. Create `src/lib/templates/data-mapper.ts`:
       - Export `mapAssessmentToTemplate(templateId: TemplateId, scoreData: ScoreResult, userEmail: string): TemplateData`
       - Extract category score for the template's matching sub-category from breakdown
       - Derive `gaps` from missingControls filtered to the template's sub-category
       - Derive `strengths` from categories with score >= 7.5 in the relevant area
       - Derive `recommendations` from missingControls recommendations
       - Set familyName from email prefix (or "Your Family" fallback)
       - Set assessmentDate from current date formatted as "Month Day, Year"
       - Determine categoryRiskLevel using getRiskLevel from scoring.ts

    All data mapping must produce plain string/number values suitable for docxtemplater placeholders (no objects, no undefined — use empty strings as fallback).
  </action>
  <verify>
    Run `npx tsc --noEmit` to verify types compile. Verify docxtemplater and pizzip are in package.json.
  </verify>
  <done>

  </done>

  <action>
    1. Create `src/lib/templates/generator.ts`:
       - Export `generateTemplate(templateId: TemplateId, data: TemplateData): Buffer`
       - Load .docx template from `templates/` directory using `path.join(process.cwd(), 'templates', templateId + '.docx')`
       - Use PizZip to unzip, Docxtemplater to render with `{ paragraphLoop: true, linebreaks: true }`
       - Return generated buffer as `doc.getZip().generate({ type: 'nodebuffer' })`
       - Export `getAvailableTemplates(): TemplateMetadata[]` returning TEMPLATE_REGISTRY

    2. Create 7 .docx template files programmatically in `templates/` directory.
       Since we cannot create real .docx files directly, create them using docxtemplater in a setup script approach:
       - Create a helper at `src/lib/templates/create-templates.ts` that programmatically builds minimal .docx files using PizZip
       - Each template should have docxtemplater placeholders: {familyName}, {assessmentDate}, {overallScore}, {riskLevel}, {categoryScore}, {categoryRiskLevel}
       - Loop sections with {#gaps}{description} - {severity} Priority\n{recommendation}{/gaps}
       - Loop sections with {#recommendations}{.}{/recommendations}
       - Run this script once via `npx tsx src/lib/templates/create-templates.ts` to generate the .docx files

       Template content for each of the 7 policies:
       a) **Decision-Making Authority Policy**: Family governance structure, voting rights, quorum requirements, conflict resolution procedures
       b) **Access Controls Policy**: Asset access tiers, approval workflows, information sharing protocols, digital access management
       c) **Trust & Estate Governance Policy**: Trustee responsibilities, beneficiary rights, distribution procedures, trust review schedule
       d) **Succession Planning Policy**: Leadership transition plan, mentorship requirements, capability assessment, timeline
       e) **Behavior Standards Policy**: Family code of conduct, social media guidelines, public representation rules, consequence framework
       f) **Family Business Governance Policy**: Board structure, family employment policies, compensation transparency, conflict of interest rules
       g) **Documentation & Records Policy**: Record retention schedule, access protocols, update frequency, responsible parties

    3. Create GET endpoint at `/api/templates/[id]` where `id` is assessmentId:
       - Query param `template` specifies which template (e.g., `?template=decision-making-authority`)
       - Query param `all=true` returns JSON list of available templates (for UI to enumerate)
       - Auth check, ownership check (same pattern as PDF route)
       - Load PillarScore, map to TemplateData using data-mapper
       - Generate .docx using generator
       - Return with headers:
         - Content-Type: application/vnd.openxmlformats-officedocument.wordprocessingml.document
         - Content-Disposition: attachment; filename="{template-name}-policy.docx"
       - If `template` param missing or invalid, return 400 with available template IDs

    IMPORTANT: Template .docx files must exist on disk before generator can use them. The create-templates script must be run as part of setup.
  </action>
