  <action>
Add a new function `calculateCustomizedPillarScore` to `src/lib/assessment/scoring.ts`. Do NOT modify the existing `calculatePillarScore` — add alongside it for backward compatibility.

```typescript
export function calculateCustomizedPillarScore(
  answers: Record<string, unknown>,
  pillar: Pillar,
  allQuestions: Question[],
  visibleQuestionIds: string[],
  emphasisMultipliers: Record<string, number>  // subcategoryId -> multiplier (e.g., 1.5)
): ScoreResult
```

Implementation mirrors `calculatePillarScore` but with one key difference:
- When accumulating subcategory scores into the pillar total, multiply each subcategory's weight by its emphasis multiplier: `totalWeightedScore += categoryScore * subCategory.weight * multiplier`
- Also adjust `totalWeight += subCategory.weight * multiplier` to maintain proper weighted average
- The multiplier comes from `emphasisMultipliers[subCategory.id]` with fallback to `1.0`

This means emphasized subcategories have more influence on the final pillar score without changing individual question scores. The per-question scoring remains unchanged — only the aggregation weights shift.

IMPORTANT: The existing `calculatePillarScore` function must remain unchanged. The new function is an extension that wraps the same logic with multiplier support.

Add tests to `src/lib/assessment/scoring.test.ts`:
- Test that `calculateCustomizedPillarScore` with all multipliers = 1.0 produces identical results to `calculatePillarScore`
- Test that a subcategory with 1.5x multiplier has proportionally more influence on final score
- Test that emphasis multipliers don't change individual category scores (only their contribution to the weighted total)
- Test with empty emphasisMultipliers (should behave like standard scoring)
