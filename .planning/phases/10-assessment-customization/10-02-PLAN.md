Modify question flow to filter questions based on customization config.

**Key insight:** The existing `useAssessmentNavigation` hook uses `getVisibleQuestions` from branching.ts which already filters by branching rules and profile conditions. We need to add an additional filter layer for subcategory visibility.

Modify `src/lib/hooks/useAssessmentNavigation.ts`:
- Add an optional `visibleSubCategories?: string[]` parameter (or accept it via a new hook/context)
- Before applying branching logic, pre-filter `allQuestions` to only those whose `subCategory` is in `visibleSubCategories`
- If `visibleSubCategories` is undefined or empty array, use all questions (backward compat)
- This ensures navigation, progress, and question display all respect the customization

**Approach for passing customization to the question page:**

The question page is a client component. To avoid prop drilling from server:
1. In the question page component, add the same `useQuery(['assessment-customization'])` fetch used in the hub
2. Extract `visibleSubCategories` from the config
3. Pass to `useAssessmentNavigation`
4. The hook uses these to pre-filter `allQuestions` before branching logic

Modify `src/app/(protected)/assessment/[pillarSlug]/[questionIndex]/page.tsx`:
- Add `useQuery(['assessment-customization'])` to fetch config (same endpoint as hub)
- Extract `visibleSubCategories` from config
- Pass to `useAssessmentNavigation(pillarSlug, questionIndex, { visibleSubCategories })`
- While config is loading, show existing loading state (questions load after config resolves)
- Progress display automatically updates because navigation hook uses filtered question set

Modify `useAssessmentNavigation` hook signature:
