**src/lib/actions/pipeline-actions.ts** - Server actions for pipeline data:

Create `getClientPipelineData` server action following the pattern in `src/lib/actions/advisor-actions.ts`:
- Call `requireAdvisorRole()` and `getAdvisorProfileOrThrow(userId)`
- Call `getClientPipeline(profile.id)` and `getPipelineMetrics(clients)`
- Return `{ success: true, data: { clients, metrics, profile } }` or `{ success: false, error: string }`

Create `addDocumentRequirement` server action:
- Accepts `{ clientId: string, name: string, description?: string }`
- Validates with Zod: name min 1 char max 100, description max 500
- Verifies advisor owns the client assignment
- Creates DocumentRequirement via Prisma
- Revalidates `/advisor/pipeline`
- Returns created requirement

Create `removeDocumentRequirement` server action:
- Accepts `requirementId: string`
- Verifies advisor owns the requirement
- Deletes the DocumentRequirement
- Revalidates `/advisor/pipeline`

**src/app/api/advisor/status-stream/route.ts** - SSE endpoint:

Create a GET route handler that:
- Authenticates the advisor using the same auth pattern (`requireAdvisorRole`)
- Creates a `ReadableStream` with `TextEncoder`
- Sends an initial `connected` event with timestamp
- Sets up a polling interval (every 30 seconds) that:
  - Fetches `getClientPipeline(advisorProfileId)`
  - Sends a `pipeline_update` event with the full client array as JSON
- Cleans up interval on stream close
- Returns `Response` with headers: `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `Connection: keep-alive`

SSE event format:
```
event: pipeline_update
data: {"clients": [...], "timestamp": "ISO8601"}
```

Note: Use polling-based SSE (not database triggers) for simplicity. 30-second interval matches the stale-while-revalidate strategy from research.

**src/lib/pipeline/hooks.ts** - Client-side hooks (mark with `"use client"`):

Create `usePipelineUpdates` hook:
- Takes initial `PipelineClient[]` data from SSR
- Maintains local state initialized from SSR data
- Opens `EventSource` to `/api/advisor/status-stream`
- On `pipeline_update` event, parses JSON and updates local state (parse dates with `new Date()`)
- Handles reconnection: `EventSource` has built-in reconnection, but also handle `onerror` to set a `connected` state
- Returns `{ clients, connected, lastUpdated }`
- Cleanup: close EventSource on unmount

Create `usePipelineFilters` hook:
- Manages filter state: `stage`, `search`, `sortBy`, `sortDir`
- Takes `PipelineClient[]` and returns filtered/sorted array
- Filter by stage: match `client.stage === selectedStage` (if set)
- Filter by search: match client.name or client.email (case-insensitive)
- Sort by selected column using `getStageOrder` for stage sorting
- Uses `useMemo` for performance
