**src/lib/pipeline/queries.ts** - Add `getClientDetail` function:

Create `getClientDetail(advisorProfileId: string, clientId: string)`:
- Verify advisor-client assignment exists (multi-tenant isolation)
- Fetch client User data (name, email, createdAt)
- Fetch invitation data: InviteCode where createdBy = advisorProfileId and prefillEmail = client.email
  - Include sentAt (createdAt), status, statusUpdatedAt
- Fetch all IntakeInterviews for client with response counts
- Fetch all Assessments for client with scores
- Fetch DocumentRequirements where advisorId and clientId
- Compute current stage and progress using existing functions
- Build a timeline of events from all data sources:
  - Each event: `{ stage: ClientWorkflowStage, label: string, date: Date, detail?: string }`
  - Events derived from: invitation creation, invitation status changes (statusUpdatedAt), intake start/completion, assessment start/completion
  - Sort chronologically

Return type `ClientDetail`:
```typescript
export type ClientDetail = {
  client: PipelineClient;
  timeline: WorkflowEvent[];
  documentRequirements: {
    id: string;
    name: string;
    description: string | null;
    required: boolean;
    fulfilled: boolean;
    fulfilledAt: Date | null;
    createdAt: Date;
  }[];
  intakeDetails: {
    status: string;
    responseCount: number;
    totalQuestions: number;
    submittedAt: Date | null;
  } | null;
  assessmentDetails: {
    status: string;
    score: number | null;
    riskLevel: string | null;
    completedAt: Date | null;
    pillarScores: { pillar: string; score: number; riskLevel: string }[];
  } | null;
};

export type WorkflowEvent = {
  stage: ClientWorkflowStage;
  label: string;
  date: Date;
  detail?: string;
};
```

Add these types to `src/lib/pipeline/types.ts`.
