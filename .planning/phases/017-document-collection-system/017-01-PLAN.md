1. Extend the existing DocumentRequirement model in prisma/schema.prisma with file storage fields:
   - fileKey String? (S3 object key)
   - fileName String? (original file name for display)
   - fileSize Int? (bytes)
   - fileMimeType String? (validated MIME type)
   Run `npx prisma db push` after schema changes.

2. Create src/lib/documents/types.ts with:
   - ALLOWED_FILE_TYPES constant: { 'application/pdf': ['.pdf'], 'image/png': ['.png'], 'image/jpeg': ['.jpg', '.jpeg'] }
   - MAX_FILE_SIZE constant: 10 * 1024 * 1024 (10MB)
   - DocumentUploadRequest type: { requirementId: string, fileName: string, fileType: string, fileSize: number }
   - DocumentUploadResponse type: { signedUrl: string, key: string }

3. Create src/lib/documents/validation.ts with validateFileUpload function:
   - Checks fileType against ALLOWED_FILE_TYPES keys
   - Checks fileSize against MAX_FILE_SIZE
   - Returns { valid: true } or { valid: false, error: string }

4. Create src/lib/documents/s3.ts with:
   - S3Client initialization using env vars (AWS_REGION, AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY)
   - Initialize client lazily at runtime (not module load) to avoid build errors when env vars missing, same pattern as Resend in src/lib/email.ts
   - generateUploadUrl(userId: string, requirementId: string, fileName: string, fileType: string): returns { signedUrl, key } where key = `documents/${userId}/${requirementId}/${Date.now()}-${fileName}`
   - generateDownloadUrl(key: string): returns presigned GET URL with 1-hour expiry
   - Install @aws-sdk/client-s3 and @aws-sdk/s3-request-presigner

Install dependencies: `npm install react-dropzone @aws-sdk/client-s3 @aws-sdk/s3-request-presigner`

<task type="auto">
  <name>Task 2: Create presigned URL API routes and document server actions</name>
  <files>
    src/app/api/documents/upload-url/route.ts
    src/app/api/documents/confirm/route.ts
    src/lib/actions/document-actions.ts

  <action>
1. Create src/app/api/documents/upload-url/route.ts (POST):
   - Authenticate user via auth() from @/lib/auth
   - Accept JSON body: { requirementId, fileName, fileType, fileSize }
   - Validate with validateFileUpload
   - Verify the requirement exists and belongs to the authenticated user (clientId === session.user.id) — this is the multi-tenant check
   - Verify requirement is not already fulfilled
   - Call generateUploadUrl to get presigned URL
   - Return { signedUrl, key } with 200

2. Create src/app/api/documents/confirm/route.ts (POST):
   - Authenticate user via auth()
   - Accept JSON body: { requirementId, key, fileName, fileSize, fileMimeType }
   - Verify the requirement exists and clientId === session.user.id
   - Update DocumentRequirement: set fulfilled=true, fulfilledAt=new Date(), fileKey=key, fileName, fileSize, fileMimeType
   - Return updated requirement with 200

3. Create src/lib/actions/document-actions.ts with server actions:
   - getClientDocumentRequirements(): authenticate as client, fetch all DocumentRequirement records for the current user with advisor's firmName and logoUrl included. Returns grouped by advisor.
   - confirmDocumentUpload(requirementId: string, fileMetadata: { key, fileName, fileSize, fileMimeType }): same logic as confirm route but as server action for use in client components. Validates ownership, updates record.
   - getDocumentDownloadUrl(requirementId: string): authenticate, verify ownership, generate presigned GET URL for the file.

Follow the same patterns as src/lib/actions/pipeline-actions.ts for error handling (try/catch, return { success, data/error }).
Follow the same auth patterns as pipeline-actions.ts using @/lib/auth for session checks.
  </action>

  </done>
