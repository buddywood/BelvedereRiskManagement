  <action>
    **ScoreBadge.tsx** (server component, no "use client"):
    A small component that displays a governance score with risk-level coloring.
    Props: `score: number | null`, `riskLevel: string | null`.
    - If score is null: show "No assessment" in muted text.
    - Otherwise: show score formatted as `{score.toFixed(1)}` with a colored dot indicator:
      - LOW: green (text-emerald-600, bg-emerald-50)
      - MEDIUM: amber (text-amber-600, bg-amber-50)
      - HIGH: orange (text-orange-600, bg-orange-50)
      - CRITICAL: red (text-red-600, bg-red-50)
    - Include dark mode variants for all colors.

    **MetricsCards.tsx** (server component):
    Props: `metrics: DashboardMetrics` (from `src/lib/dashboard/types.ts`).
    Render a responsive grid of 4 metric cards (same `hero-surface` pattern as existing advisor page.tsx):
    - Total Families: `metrics.totalClients`
    - Average Score: `metrics.averageScore?.toFixed(1) ?? 'N/A'` with "/10" suffix
    - High Risk: `metrics.highRiskCount` (use orange/red text if > 0)
    - Assessed: `metrics.assessedCount` of `metrics.totalClients`
    Grid: `grid-cols-2 sm:grid-cols-4 gap-4`.

    **GovernanceTable.tsx** ("use client"):
    Props: `clients: DashboardClient[]` (from `src/lib/dashboard/types.ts`).
    Use TanStack React Table with these columns:

    1. **Family Name** (accessorKey: 'name'): Display name or "Unnamed Family". Enable sorting.
    2. **Governance Score** (accessorFn from latestScore): Render using ScoreBadge component. Sort by score value (null sorts last).
    3. **Risk Level** (accessorFn from latestScore.riskLevel): Render as colored Badge component (use existing shadcn Badge). Sort by severity order (CRITICAL > HIGH > MEDIUM > LOW > null).
    4. **Assessments** (accessorKey: 'assessmentCount'): Simple number display.
    5. **Last Assessment** (accessorFn from latestAssessmentDate): Format with date-fns `format` as "MMM d, yyyy" or "No assessments" if null.

    Enable sorting with `getSortedRowModel()` from TanStack. Default sort by governance score descending (highest risk first — null scores at top so advisors see unassessed families).

    Table styling: Use existing shadcn `<table>` markup patterns (`w-full`, `rounded-md border`, `px-4 py-2` cells). Use Tailwind classes consistent with existing Card/Table patterns in the codebase.

    Responsive: On mobile/tablet (below `lg`), hide the "Assessments" and "Last Assessment" columns using TanStack column visibility API. Set initial column visibility based on `typeof window !== 'undefined' && window.innerWidth < 1024`.

    Empty state: If `clients.length === 0`, show a centered message "No clients assigned yet. Contact your administrator to get started." matching existing empty state pattern from advisor page.tsx.
