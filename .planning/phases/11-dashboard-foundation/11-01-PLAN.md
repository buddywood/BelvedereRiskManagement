<task type="auto">
  <name>Task 2: Create advisor-scoped dashboard query functions</name>
  <files>src/lib/dashboard/queries.ts</files>
  <action>
    Create `src/lib/dashboard/queries.ts` with `"server-only"` import at the top (same pattern as `src/lib/data/advisor.ts`).

    Import prisma from `@/lib/db` and dashboard types from `./types`.

    **Function `getAdvisorDashboardClients(advisorProfileId: string): Promise<DashboardClient[]>`:**

    1. Query `prisma.clientAdvisorAssignment.findMany` filtered by `advisorId: advisorProfileId` and `status: 'ACTIVE'`.
    2. Include `client` with nested include of `assessments` where `status: 'COMPLETED'`, ordered by `completedAt: 'desc'`, including `scores` (ordered by `calculatedAt: 'desc'`, take 1).
    3. Map results to `DashboardClient[]`:
       - `latestScore`: From the first assessment's first score — extract `score`, `riskLevel`, `calculatedAt`, `breakdown`. Null if no completed assessments.
       - `assessmentCount`: Length of the client's completed assessments array.
       - `latestAssessmentDate`: `completedAt` from the first assessment, or null.

    IMPORTANT: Do NOT eagerly load all assessments. Only load completed ones. The `take` on scores (1) prevents N+1 since it's a nested include, not a separate query.

    **Function `getDashboardMetrics(clients: DashboardClient[]): DashboardMetrics`:**

    This is a pure function (no DB query) that computes metrics from the client array:
    - `totalClients`: clients.length
    - `assessedCount`: clients with non-null `latestScore`
    - `averageScore`: Average of all non-null `latestScore.score` values. Null if no assessed clients.
    - `highRiskCount`: Count of clients where `latestScore.riskLevel` is `HIGH` or `CRITICAL`.

    **Function `getRiskDistribution(clients: DashboardClient[]): RiskDistribution`:**

    Pure function. Counts clients in each risk band based on `latestScore.riskLevel`. Clients with no score are excluded from the distribution.

    All three functions must be exported.

    Security note: The `advisorProfileId` parameter comes from `getAdvisorProfileOrThrow()` which is already authenticated. The `ClientAdvisorAssignment` filter ensures row-level isolation — advisors only see their assigned clients.
  </action>
